Check a boolean value. If true, run optional code. If false, skip the optional code.

Syntax: 
    boolean condition = true;
    if(condition){
        System.out.println("This line will be executed");
    }

Output: This line will be executed

Since the boolean value in the if statement is true, the code in the {}'s will be executed. 

    boolean condition = false;
    if(condition){
        System.out.println("This line will not be executed");
    }
    System.out.println("Control skips to here when the condition is false");

Output: Control skips to here when the condition is false

Since the boolean value in the if statement is false, the code in the {}'s will not be executed.

== Boolean Expressions

The boolean value in the if statement can be a boolean expression that evaluates to a single boolean value.

Common boolean comparisons:

  • Greater than: >
  • Less than: <
  • Equal: ==
  • Greater than or equal: >=
  • Less than or equal: <=
  • Not equal: !=

    boolean comparison = 5 > 10;
    System.out.println(comparison);

Output: false

At most 1 block is executed in an if/else section.

    int number = 45;
    if(number < 30){
        System.out.println("number is less than 30");
    } else if(number < 40){
        System.out.println("number is less than 40");
    } else if(number < 50){
        System.out.println("number is less than 50");
    } else if(number < 60){
        System.out.println("number is less than 60");
    } else {
        System.out.println("number is greater than, or equal to, 60");
    }

Output:number is less than 50


The last condition number < 60 is true, but is never checked since a previous condition was true. Once a true condition is found the corresponding block of code is executed and control moves to the end of the entire group of conditionals.

{#== Nested Ifs#} == Random Number Generator (RNG)

Math.random() will return a pseudo-random double between 0.0 and 1.0. 

    if(Math.random() > 0.5){
        System.out.println("There is a 50% chance this line will be printed");
    }

Each time the above is executed there will be a 50% chance that the String value will be printed to the console.

What if we don't want a value in the range 0.0-1.0? What if we want a value from 0.0-100.0? We'll multiply the result!

    double randomNumber = Math.random()*100;
    System.out.println("The random value is: " + randomNumber);

The output will be a random double between 0.0 and 100.0

{#== Switch/Case#} {##} {#Might come later, if ever.#} == Block and Scope

Any time a section of code is surrounded by {braces} we call this section a code block. We saw code blocks corresponding to if statements and we will see them in loops as well. Code blocks are most commonly used in control flow to tell the program which section of code to run after checking a conditional. When the conditional of an if statement is true, the following block of code is executed. When the conditional of an if statement is false, the following block of code is skipped.

Variables created inside a block or code are destroyed at the end of that block. 

    int number = 5;
    if(number > 3){
        double secondNumber = 7.43; // create a double variable and name it secondNumber
    } // secondNumber is destroyed here since it was created inside the block that is ending
    System.out.println(secondNumber); // This will cause an error since secondNumber has been destroyed

This code will cause a compiler error (the computer cannot understand how to run the code).

== While Loop

Repeat a block of code while a condition is true. Once the condition is false, stop repeating the code.

Example: We want to set a thermostat to a desired temperature, but we can only increase the temperature 1 degree at a time. 

    int temperature=65;
    while(temperature<70){
        System.out.println("Temperature set to: " + temperature);
        temperature = temperature+1;
    }
    System.out.println("Final Temperature: " + temperature);

Output:

Temperature set to: 65
Temperature set to: 66
Temperature set to: 67
Temperature set to: 68
Temperature set to: 69
Final Temperature: 70

This could be done without a loop: 

    int temperature=65;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    System.out.println("Final Temperature: " + temperature);

This is repetitive, but it isn't too bad. However, this code is much less flexible than our code that uses a while loop. Let's assume we want o set the temperature to 75 instead of 70. With a while loop our example becomes: 

    int temperature=65;
    while(temperature<75){
        System.out.println("Temperature set to: " + temperature);
        temperature = temperature+1;
    }
    System.out.println("Final Temperature: " + temperature);

Output:

Temperature set to: 65
Temperature set to: 66
Temperature set to: 67
Temperature set to: 68
Temperature set to: 69
Temperature set to: 70
Temperature set to: 71
Temperature set to: 72
Temperature set to: 73
Temperature set to: 74
Final Temperature: 75

We barely had to change any code to have a large impact on the output. If we want the same output in the example without a loop we will have to change much more code: 

    int temperature=65;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;
    temperature = temperature+1;

This code is less flexible and less readable than the code using a loop. What's worse is that we can't even write this code if we don't know how many degrees the temperature needs to be increased by. What if we want to write code that sets the temperature to 70 degrees regardless of the current temperature setting? How many times do we write "temperature = temperature+1;"?

  • Condition is checked before each iteration. If the condition is false when the loop is first reached it will never execute.
  • Condition should be false eventually! If the condition is never false the loop with run forever!

{#Guessing game. Track number of guesses. While the user hasn't guessed correctly, ask for another guess.#} {#Print number of guesses and correct answer. [Need user input]#} == For Loops

Has the same power as while loops with different syntax. 

    for([initialization]; [condition]; [increment]){
        // body
    }

Where a while loop only needs a condition, for loops need 3 separate expressions, initialization code to be executed before the loop starts, the condition to be checked before each iteration, and an increment statement executed after each iteration.

  • Initialization: A statement that is executed before the loop begins. Initialization is only executed once.
  • Condition: The boolean expression that controls the number of iterations. The loop will run until the condition is false.
  • increment: Code to be executed each time the end of the loop is reached. This most commonly alters a value that is used in the condition to control the number of times the loop is executed. 
    for(int i=0; i<3; i++){
        System.out.println("The value stored in i is " + i);
    }

Output:

The value stored in i is 0
The value stored in i is 1
The value stored in i is 2

In this example we are initializing an int variable names i and setting it to 0 before the loop starts, Checking the condition "i<3" to control the loop, and increasing i by 1 after each iteration. This code is equivalent to the following. 

    {
        int i=0;
        System.out.println("The value stored in i is " + i);
        i++;
        System.out.println("The value stored in i is " + i);
        i++;
        System.out.println("The value stored in i is " + i);
        i++;
    }

Note that since the variable i is declared in the loop it will fall out of scope (be destroyed) when the loop is finished. 

    for(int i=3; i<5; i++){
        System.out.println(i*i);
    }

Output:

9
16

== Continue and Break

When you want to make an exception to the loops condition.

  • Continue: Skips the rest of the current iteration and moves to the increment statement
  • Break: Exit the loop entirely even if the condition is still met
    for(int i=0; i<7; i++){
        if(i == 4){
            continue;
        }
        System.out.println("The value stored in i is " + i);
    }

Output:

The value stored in i is 0
The value stored in i is 1
The value stored in i is 2
The value stored in i is 3
The value stored in i is 5
The value stored in i is 6

When i is 4 the continue is executed which skips to the end of the block without printing to the screen, then the continues as usual. 

    for(int i=0; i<7; i++){
        if(i == 4){
            break;
        }
        System.out.println("The value stored in i is " + i);
    }

Output:

The value stored in i is 0
The value stored in i is 1
The value stored in i is 2
The value stored in i is 3

When i is 4 the break is executed which immediately exits the loop even though i<7 is still true.
